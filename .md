#Leave Request Flow Enhancement
-Add a “Re-request / Retract Leave” option to existing leave requests.

##Requirement:
-If an employee’s plans change after submitting a leave request, they should be able to modify the request without creating a new one.

##Scenario Example:
-An employee requested leave for 17 March, and the request was approved by the admin.
-Later, the employee decides to come to the office and no longer needs the leave.
-The employee should be able to send a re-request (retract/modify request) on the same leave entry.

##Expected Behavior:
-The employee selects Re-request / Retract on the approved leave request.
-The system sends the update request to the admin for approval.

##Once approved:
-The leave entry is cancelled or updated.
-The employee’s remaining leave balance is recalculated.
-Attendance for that date should not be marked as leave/absent.

##Additional Rules:
-Only the employee who created the request can send a re-request.
-The admin must approve the change before it affects attendance or leave balance.
-The system should maintain history of the original request and the re-request action.**
